NetBeans pour Wakanda Linux

Installer NetBeans

  • Se rendre sur la page Oracle de téléchargement de JavaSE et choisir JDK 7u45 & NetBeans 7.4 (ou plus récent). Si c'est trop long, prendre le script dans smb://srv-4d/SOFT/Linux/NetBeans/
  • Lancer le script téléchargé, par exemple jdk-7u45-nb-7_4-linux-x64.sh, et choisir des répertoires d'installation appropriés, par exemple ~/NetBeansBundle/jdk et ~/NetBeansBundle/nb

Configurer NetBeans

  • Allouer plus de mémoire à la jvm en modifiant le fichier ~/NetBeansBundle/nb/bin/netbeans ; à la fin de la fonction heap_size, modifier la ligne précédant le return avec max_heap_size=4096 pour accorder 4Go à la machine virtuelle java.
  • Lancer Netbeans avec la commande modifiée : ~/NetBeansBundle/nb/bin/netbeans ; NetBeans se lance en utilisant le jdk téléchargé (ce qu'un ps doit confirmer).
  • Verifier la connexion internet en allant dans Tools -> Options -> General, bouton Test connection ; au besoin, configurer le proxy manuellement.
  • Installer le plugin C++ en allant dans Tools -> Plugins, onglet Available Plugins, sélectionner C/C++ et cliquer Install
  • Installer le plugin (non officiel) Perforce en téléchargeant le fichier npm le plus récent de netbeans-perforce, par exemple org-wonderly-netbeans-perforce-2.2.2.nbm, puis en sélectionnant ce fichier depuis Tools -> Plugins, onglet Downloaded, bouton Add Plugins… Une fois le plugin ajouté, cliquer Install. Le client en ligne de commande preforce doit être préalablement configuré.
  • Configurer quelques options de formatage en allant dans Tools -> Options -> Editor, onglet Formating
    • Sélectionner Language: All Languages
    • Sélectionner Category: Tabs And Indents
    • Décocher Expand Tabs to Spaces
    • Choisir Number of Spaces per Indent: 4
    • Choisir Tab Size: 4
    • Choisir Right Margin: 0
    • Cliquer OK
  • Eventuellement, choisir un thème plus cohérent avec Linux en allant dans Tools -> Options -> Miscellaneous, onglet Windows, menu déroulant Preferred look and feel: GTK+

Développer et débugger Wakanda

  • Eventuellement, se rendre dans //depot/Wakanda/main/Common/CMake et lancer ./ResetBuild.sh
  • Ouvrir le projet Wakanda en allant dans File -> Open project et naviguer jusqu'au répertoir //depot/Wakanda/main/Common/NetBeans et choisir Wakanda
  • NetBeans charge le projet et commence l'indexation des sources en tâche de fond, ce qui le rend un peu apathique.
  • Lorsque Netbeans demandera s'il faut traiter les fichiers .ci comme du C++, répondre oui.
  • Choisir EnterpriseDbg64 dans le menu déroulant Set Project Configuration
  • Lancer une compilation en cliquant l'icône Clean and Build Project ; un onglet Output s'ouvre pour afficher l'avancement des opérations (moins de 6 minutes en principe).
  • Aller dans Debug -> New Breakpoint et entrer Function : main
  • Cliquer Debug Project ; le debuggeur prend son temps puis se lance, le programme s'arrête dans main…
